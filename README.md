# 股票失误反省录

记录每一次误操作，让下一次少犯错。

一个极简微信小程序，用来记录股票交易中的误操作。数据只保存在本地，通过亏损比例和影响仓位折算实际损失，帮助用户用数据复盘反复犯错的模式。

## 功能

- 新增误操作：日期、亏损比例、影响仓位、标的、失误原因、复盘备注。
- 失误原因下拉：追涨买入、恐慌割肉、卖飞、频繁交易、重仓赌单票、没按计划止损、没按计划止盈、犹豫错过、其他、自定义。
- 月历概览：按月显示每天是否有误操作，点击日期查看当天详情。
- 趋势统计：近 3 个月按周展示；自定义范围默认近 1 年，并按跨度自动按天、周或 30 天分桶。
- 原因统计：每个统计范围内按失误原因汇总次数和损失，按损失从高到低排序。
- 图表详情：点击柱子弹出该时间段的记录明细。
- 编辑删除：概览日期详情中支持编辑和删除单条记录。
- 历史查询：默认近 1 年，可自定义开始和结束日期。
- 备份恢复：支持导出 JSON，支持导入合并或覆盖。

## 统计口径

- 实际损失 = 亏损比例 × 影响仓位。
- 统计以满仓口径展示。例如亏损 `10%`、影响仓位 `30%`，实际损失为 `-3%`。
- 旧数据或导入数据没有影响仓位时，默认按 `100%` 计算。
- 导入时影响仓位会限制在 `0-100`，避免异常数据放大损失。

## 数据与备份

- 不依赖后台服务器，所有记录保存在微信小程序本地缓存。
- 导出 JSON 后会通过微信文件分享发送，建议转发到文件传输助手、个人聊天或网盘。
- 换手机流程：旧手机导出 JSON，新手机打开小程序后在“备份”页选择导入合并或导入覆盖。
- JSON 是完整备份格式，适合恢复数据。

## 开发使用

1. 打开微信开发者工具。
2. 导入项目目录：`/Users/bingliang/work/proj_TradeRecord`。
3. 使用 `project.config.json` 中的 AppID，或按需改成自己的测试号。
4. 编译后从“新增”页记录数据，在“概览”和“历史”页查看统计。

## 技术说明

- 原生微信小程序，无后台服务。
- 本地存储使用 `wx.setStorageSync` / `wx.getStorageSync`。
- 备份导入导出使用 JSON 文件。
- 已启用组件按需注入：`lazyCodeLoading: "requiredComponents"`。
